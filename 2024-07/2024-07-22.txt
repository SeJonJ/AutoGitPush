제목: 제목: JEP 472: JDK 24에서 JNI 사용을 제한하기 위해 준비하기

JEP 472는 JDK 24에서 JNI(Java Native Interface)의 사용을 제한하기 위한 조치를 취할 것임을 나타내는 명령입니다. JNI는 Java와 네이티브 코드 간의 통신을 허용하는 인터페이스로, 자바 언어에서 네이티브 코드와 상호작용할 수 있게 해주는 역할을 합니다. 그러나 JNI는 보안 문제와 성능 문제 등의 이유로, 향후 버전의 JDK에서 제한될 가능성이 높습니다.

JEP 472는 이러한 문제를 해결하기 위해 JNI 사용을 제한하고, 안정적이고 안전한 대체 솔루션을 제공할 것입니다. 이를 통해 Java 개발자들은 JNI를 사용하지 않는 코드를 작성하도록 유도되며, 보안과 성능에 대한 우려를 줄일 수 있습니다.

JEP 472의 구현은 JDK 24에서 시작되고, 향후 버전에서는 더욱 강력한 제한을 적용할 것입니다. 이러한 변경으로 인해 Java 개발자들은 JNI를 사용하지 않도록 주의를 기울여야 합니다.
주소: https://www.infoq.com/news/2024/07/jep-472-restrict-jni-jdk-24/
설명: JEP 472, now Proposed를 대상으로 이루어지며, Java Native Interface(JNI)를 사용할 때 경고를 발생시키고 Foreign Function & Memory(FFM) API를 일관성 있는 경고로 조정하려고 합니다. 이는 미래의 릴리즈에서 JNI와 FFM API를 기본적으로 통합성을 보장하기 위해 준비를 함으로써 개발자들을 대비시킵니다.
