제목: 제목: JEP 472: JDK 24에서 JNI 사용을 제한하기 위해 준비하기

JEP 472는 JDK 24에서 JNI(Java Native Interface)의 사용을 제한하기 위한 조치를 취할 것임을 나타내는 명령입니다. JNI는 Java와 네이티브 코드 간의 통신을 허용하는 인터페이스로, 자바 언어에서 네이티브 코드와 상호작용할 수 있도록 합니다. 그러나 JNI는 보안 문제와 성능 문제 등의 이유로 최근에는 자주 사용되지 않습니다.

JEP 472는 JDK 24에서 JNI를 더 이상 사용하지 않도록 변경할 것입니다. 이를 통해 Java 개발자들은 JNI를 사용하지 않고 네이티브 코드와 상호작용하는 데 대체 기능을 찾아야 합니다. 또한, JDK 24에서 JNI의 사용을 제한함으로써 Java 플랫폼의 안정성과 보안성이 향상될 것입니다.

따라서, JEP 472는 JDK 24에서 JNI 사용을 제한하기 위해 준비하는 것으로 간주됩니다. 개발자들은 이 변경 사항에 대처하고, JNI를 사용하지 않는 대체 솔루션을 찾아야 합니다.
주소: https://www.infoq.com/news/2024/07/jep-472-restrict-jni-jdk-24/
설명: JEP 472, now Proposed를 대상으로 이동하여, Java Native Interface(JNI)를 사용할 때 경고를 발생시키고 Foreign Function & Memory(FFM) API를 일관된 경고로 조정하려고 합니다. 미래의 릴리스에서 JNI와 FFM API를 기본적으로 통합성을 보장하기 위해 준비를 합니다.
