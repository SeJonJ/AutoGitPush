제목: 제목: JEP 472: JDK 24에서 JNI 사용을 제한하기 위해 준비하기

JEP 472는 JDK 24에서 JNI(Java Native Interface)의 사용을 제한하기 위한 조치를 취할 것임을 나타냅니다. JNI는 Java와 네이티브 코드 간의 통신을 허용하는 인터페이스로, 자바 언어와 다른 프로그래밍 언어(C, C++ 등)를 연결하는 데 사용됩니다. 이 제안은 JNI의 사용을 제한하여 보안과 성능 문제를 해결하고, Java 언어와 네이티브 코드 간의 통신에 대한 표준적인 방법을 제공하기 위함입니다.

JDK 24에서 JNI 사용을 제한하려면, 먼저 JNI를 사용하는 모든 코드와 라이브러리를 확인하고, 필요한 경우 대체 방법을 찾아야 합니다. 또한, JNI를 사용하지 않는 새로운 API를 제공하여 개발자들이 JNI 대신 이러한 API를 사용할 수 있도록 해야 합니다.

JEP 472의 목표는 JDK에서 JNI 사용을 최소화하고, 안전하고 효율적인 Java 프로그래밍 방식을 제공하는 것입니다. 이를 통해, Java 개발자들은 JDK를 사용할 때 보다 더 안전하고 효율적인 코드를 작성할 수 있을 것입니다.
주소: https://www.infoq.com/news/2024/07/jep-472-restrict-jni-jdk-24/
설명: JEP 472, now Proposed to Target,는 Java Native Interface(JNI)를 사용할 때 경고를 발생시키고 Foreign Function & Memory(FFM) API를 일관된 경고로 조정하려고 합니다. 이것은 미래의 릴리스에서 JNI와 FFM API를 기본적으로 통합성을 보장하기 위해 개발자들을 준비시킵니다.
#JEP472 #JNI #FFM_API #통합성
